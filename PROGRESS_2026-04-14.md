## Setup inicial — Proxmox API

### Flujo seguido (Opción B: usuario dedicado)
Documentado paso a paso el proceso recomendado en lugar de usar `root@pam`:
1. Crear usuario `conky@pve` en Datacenter → Users.
2. Añadir permiso `PVEAuditor` sobre path `/` con Propagate.
3. Crear API token `conky@pve!conky` con Privilege Separation desmarcada.
4. Rellenar `.proxmox.env` (host=192.168.8.11, puerto 8006, token id, secret, `PVE_INSECURE=1`).

### Debugging conexión
- Primer intento falló con `STATUS=UNREACHABLE` en el cache, pero host respondía al ping.
- `curl` manual reveló `HTTP 401` → token ID incorrecto en `.env` (decía `root@pam!conky` en vez de `conky@pve!conky`).
- Tras corregir el user: `HTTP 200`, script devuelve `STATUS=OK` con 10 guests (4 running: openclaw, dokploy, pihole, cloudflared) y 2 storages (local, local-zfs).
