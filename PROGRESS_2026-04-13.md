## Implementado — Widget Proxmox

### Decisiones
- Conexión vía **API token** (PVEAPIToken header, curl + jq).
- Credenciales en archivo `.proxmox.env` (gitignored).
- Métricas: CPU/MEM/DISK del nodo, uptime, load, lista VMs/LXC (estado + cpu% + mem%), storage pools con barras.
- Posición: **banda inferior central** (x=0.30, y=0.90, ancho 0.50w).
- Refresco: 30s (vía `execi` en `conky.conf`).

### Archivos creados
- `modern/.proxmox.env.example` — plantilla de credenciales (host, puerto, token id, secret, flag insecure).
- `modern/scripts/proxmox.sh` — bash + curl + jq:
  - GET `/api2/json/nodes` → nodo, cpu, mem, disk, uptime
  - GET `/api2/json/nodes/{node}/status` → loadavg
  - GET `/api2/json/nodes/{node}/qemu` → VMs
  - GET `/api2/json/nodes/{node}/lxc` → contenedores
  - GET `/api2/json/nodes/{node}/storage` → storages
  - Salida a `/tmp/conky-proxmox.txt` formato multi-sección (`KEY=value` + `GUEST=...|...|...` + `STORE=...|...`).
  - Estados de error: `STATUS=NO_ENV|NO_JQ|UNREACHABLE|NO_NODE|OK`.

### Archivos modificados
- `modern/draw.lua`:
  - Nuevo parser `read_proxmox(ttl)` — maneja el formato multi-sección con cache TTL.
  - Nueva función `draw_proxmox(cr, x, y, w, s)` — dibuja header (nodo + métricas + uptime + load), rejilla 4×2 de guests con dots de estado (verde=VM running, ámbar=LXC running, gris=stopped), barras horizontales por storage con color según % (verde <70, ámbar 70-85, rojo >85).
  - Llamada añadida al final de `conky_main`.
- `modern/conky.conf`:
  - Añadido `${execi 30 scripts/proxmox.sh}` a `conky.text`.
- `modern/.gitignore`:
  - Añadido `.proxmox.env`.
